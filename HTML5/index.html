Targets

_blank - tell the browser to open it in a new tab or window (per user preference).
            
                Using target="_blank" gives the opening site partial access to the window.opener object via JavaScript,
                which allows that page to then access and change the window.opener.location of your page and
                potentially redirect users to malware or phishing sites.
                Whenever using this for pages you do not control, add rel="noopener" to your link to prevent the
                window.opener object from being sent with the request.
                Currently, Firefox does not support noopener, so you will need to use rel="noopener noreferrer" for
                maximum effect

_self - This opens an anchor in the same frame.
_parent - Will open the in the next level up of a frame if they were nested to inside one another
_top - This breaks outside of all the frames it is nested in and opens the link as top document in the browser window.

Classes and ids

            class Indicates the Class of the element (non-unique)
            id Indicates the ID of the element (unique in the same context)
            
                The only restrictions on the value of an id are:
                1. it must be unique in the document
                2. it must not contain any space characters
                3. it must contain at least one character
            
            
                An id and a class value must begin with a letter, which can then be followed only by:
                letters (A-Z/a-z)
                digits (0-9)
                hyphens ("-")
                underscores ("_")
                colons (":")
                periods (".")

                
                Using numbers, punctuation or special characters in the value of an id or a class may cause trouble in other
                contexts, such as CSS, JavaScript and regular expressions.
